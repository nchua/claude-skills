# Feature Implementation

Before writing any code, create a brief todo list for the requested feature. Break it down by layer:

1. **Database/Model** — models, fields, migrations, relationships
2. **API** — routes, schemas, validation, permissions
3. **Backfills** — if needed for existing data, run with nohup and log output
4. **Frontend** — HTML templates, iOS views, CSS, assets
5. **Tests** — pytest for backend, validate end-to-end
6. **Docs** — update README and relevant docs

Work through each layer in order. Commit and push after completing each layer.

## Gotchas

These are real failure patterns from production. Check each one before considering a feature complete.

1. **Untracked files break deploys.** Run `git status` before pushing — new files that aren't `git add`ed cause ModuleNotFoundError or missing template errors in production. If you created a file, it needs to be tracked.

2. **SQLAlchemy relationships are dead after commit.** Accessing a lazy-loaded relationship after `session.commit()` raises DetachedInstanceError. Either `session.refresh(obj)` after commit, use `eager loading` (joinedload/selectinload), or access everything you need *before* committing.

3. **Name collisions across layers.** A Swift struct named `Workout` and a Python model named `Workout` is fine — but two Swift structs named `WorkoutResponse` in different files causes ambiguous type errors. Check for duplicate names across the entire layer, not just the current file.

4. **Date format mismatches between layers.** Backend sends `2024-01-15T10:30:00Z`, frontend expects `2024-01-15`, iOS expects `Date()`. Pick one canonical format at the API boundary and convert at the edges. Don't let each layer invent its own format.

5. **Credentials leak into code.** Never hardcode emails, passwords, API keys, or tokens — not in seed scripts, not in tests, not in config files. Use `os.environ.get()` and placeholder values (`test@example.com`, `TestPass123!`). GitGuardian will catch it, but after it's already in git history.

6. **Timezone bugs cascade.** One naive `datetime.now()` instead of `datetime.now(timezone.utc)` can trigger 5+ follow-up fixes across serializers, comparisons, and display logic. Use UTC everywhere internally, convert to local only at the display layer.

7. **Migration conflicts from parallel work.** Multiple Alembic migrations with the same `down_revision` create "multiple heads" errors. Run `alembic heads` before creating a migration. If you see multiple heads, merge them with `alembic merge`.

8. **Don't edit generated files.** If a file is generated by a tool (xcodegen, protobuf, OpenAPI codegen), edits will be silently overwritten next time the tool runs. Edit the source config/spec instead.

9. **Fix the config, not the derived file.** When something's wrong in a generated output (Xcode project, compiled CSS, bundled JS), trace back to the source of truth and fix it there. Otherwise the "fix" disappears on next build.

10. **Sensitive data in synced storage.** Don't store auth tokens, API keys, or PII in UserDefaults, localStorage, or other synced/inspectable storage. Use Keychain (iOS), secure cookies (web), or encrypted storage. UserDefaults syncs to iCloud and is readable in plain text.
